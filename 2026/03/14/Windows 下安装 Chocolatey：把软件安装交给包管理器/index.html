Windows 下安装 Chocolatey：把软件安装交给包管理器

发表于2026-03-14|更新于2026-03-14|开发工具

为什么装 Chocolatey

在 Windows 上装软件，很多时候还是靠浏览器下载 .exe 或 .msi，然后一路点下一步。
这种方式不是不能用，但一旦机器上软件多起来，问题会很明显：

  • 装过什么很难统一管理
  • 批量重装时效率很低
  • 更新、卸载、脚本化都不顺手

Chocolatey 的价值就在这里。它把软件安装这件事收敛成了统一命令：

这件事本质上不是“换一个安装方式”，而是把 Windows 上的软件管理拉回到可重复、可脚本化、可维护的状态。

安装前先确认 3 件事

根据 Chocolatey 官方安装文档，直接从官网脚本安装前，至少要确认下面几点：

  1. 当前系统是受支持的 Windows 版本
  2. PowerShell 可用
  3. 机器上具备 .NET Framework 4.8，或者允许安装脚本自动补齐

另外还有两个实际前提：

  • 最稳妥的方式是用管理员权限打开 PowerShell
  • 如果执行策略是 Restricted，要先临时放开当前进程的限制

哪些场景里它特别值

我认为 Chocolatey 最值的不是“省掉几次点下一步”，而是下面这些场景：

新机器初始化

重装系统或者换新电脑时，可以把常用软件收成一组命令，一次性装完。

批量维护开发环境

如果一台 Windows 机器上有 Git、Node.js、7-Zip、VS Code、Python、Docker Desktop 这些常用工具，后面统一升级会比手工点更新干净得多。

需要注意的边界

Chocolatey 很强，但它不是“装了就不用思考”。至少有几个边界要清楚：

1. 不是所有包都值得盲装

官方社区仓库很方便，但包的质量还是要看维护情况。
做法应该是：

  • 先 choco search
  • 再看包页面信息
  • 再决定是不是要装

2. 管理员权限仍然很常见

Chocolatey 是包管理器，不是权限绕过工具。
很多软件安装本身就需要管理员权限，这一点不会因为换成 choco install 就消失。

3. 适合脚本化，但也更要求你对命令负责

一旦你把安装过程写进脚本，速度会快很多，但错误也会放大得更快。
所以包名、来源和命令参数都要先确认，不要靠猜。

一个足够实用的起步方式

如果只是想先把它用起来，我建议最低限度先记住这组流程：

  1. 管理员权限打开 PowerShell
  2. 检查 Get-ExecutionPolicy
  3. 执行官方安装命令
  4. 用 choco --version 验证
  5. 先试装一个低风险工具，比如 git 或 7zip
