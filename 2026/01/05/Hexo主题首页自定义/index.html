如何自定义 Hexo 首页（Butterfly 主题）

Created2026-01-05|Updated2026-01-06|博客搭建
|Post Views:

适用场景：已搭好 Hexo + Butterfly主题（例如根目录有 _config.yml 和 _config.butterfly.yml），现在想把“首页”改成更像个人主页：换 banner、改菜单、加社交图标、做打字副标题、控制文章摘要、加载自定义 CSS/JS。

0. 先搞清楚：Hexo 首页“内容”来自哪里？

很多新手会去找一个“首页.md”，但 Hexo 的首页通常不是某个单独 Markdown 文件，而是：

  • 站点配置（_config.yml）提供网站标题/作者等基础信息
  • 主题配置（_config.butterfly.yml）决定首页的布局效果（banner、菜单、社交图标、摘要展示等）
  • 主题模板（themes/butterfly/layout/）负责把“文章列表”渲染成首页页面
  • 文章本身（source/_posts/）决定首页文章卡片展示的标题、封面、摘要等

所以要“自定义首页”，大多数时候改的是：
✅ _config.yml + _config.butterfly.yml（80% 的效果）
更深度再去动：✅ Inject / ✅ 模板文件（20% 的效果）

1. 可以改哪些文件？（结合目录）

目录大概是这样：

  • /_config.yml：Hexo 站点配置
  • /_config.butterfly.yml：Butterfly 主题配置（推荐改这个）
  • /source/：
    • source/_posts/：文章都在这
    • source/images/ 或 source/img/：放你的 banner、头像等静态资源（推荐）
  • /public/：编译输出目录（不要手改，每次 hexo g 都会重建）

建议：图片/CSS/JS 尽量放在根目录 source，不要放主题目录里，避免以后升级主题被覆盖。

4. _config.yml调整

为了保证能够正常展示自定义主页，并且能够将主页和需要展示的文章列表页进行恰当的链接，需要对_config.yml文件进行调整。主要调整内容如下：

index_generator:
path: ‘/articles/‘
per_page: 10
order_by: -date

index_generator下的path需要调整为一个新的空文件夹的名字，该文件夹下没有内容，这样的话，hexo才能够将原来的主页渲染到这个路径下。
自定义主页需要调整的超链接也需要填入当前路径，当然可以是相对路径，从/source/下算起即可。
