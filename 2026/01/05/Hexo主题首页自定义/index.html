如何自定义 Hexo 首页（Butterfly 主题）

Created2026-01-05|Updated2026-01-05|博客搭建
|Post Views:

适用场景：已经搭好 Hexo + Butterfly（例如根目录有 _config.yml 和 _config.butterfly.yml），现在想把“首页”改成更像个人主页：换 banner、改菜单、加社交图标、做打字副标题、控制文章摘要、加载自定义 CSS/JS。

0. 先搞清楚：Hexo 首页“内容”到底来自哪？

很多新手会去找一个“首页.md”，但 Hexo 的首页通常不是某个单独 Markdown 文件，而是：

  • 站点配置（_config.yml）提供网站标题/作者等基础信息
  • 主题配置（_config.butterfly.yml）决定首页的布局效果（banner、菜单、社交图标、摘要展示等）
  • 主题模板（themes/butterfly/layout/）负责把“文章列表”渲染成首页页面
  • 文章本身（source/_posts/）决定首页文章卡片展示的标题、封面、摘要等

所以你要“自定义首页”，大多数时候改的是：
✅ _config.yml + _config.butterfly.yml（80% 的效果）
更深度再去动：✅ Inject / ✅ 模板文件（20% 的效果）

1. 你应该改哪些文件？（结合你的目录）

你的目录大概率是这样：

  • /_config.yml：Hexo 站点配置
  • /_config.butterfly.yml：Butterfly 主题配置（推荐改这个）
  • /source/：
    • source/_posts/：文章都在这
    • source/images/ 或 source/img/：放你的 banner、头像等静态资源（推荐）
  • /public/：编译输出目录（不要手改，每次 hexo g 都会重建）

建议：自己的图片/CSS/JS 尽量放在根目录 source，不要放主题目录里，避免以后升级主题被覆盖。

2. 自定义顶部大图（首页 Banner）

Butterfly 把“首页顶部图”叫做 index_img。你可以用在线图，也可以用本地图。

2.1 使用本地图片（推荐）

把图片放到：source/img/home.jpg
然后在 _config.butterfly.yml 配置：

# 顶部图
index_img: /img/home.jpg
