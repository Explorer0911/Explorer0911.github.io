如何自定义 Hexo 首页（Butterfly 主题）

发表于2026-01-05|更新于2026-03-12|博客搭建
|浏览量:

先明确首页到底承担什么职责

Hexo 默认更偏向“文章流首页”，但个人博客未必一定要这样。

如果你想让博客更像个人主页，比较稳的做法是：

  • / 作为自定义首页
  • /articles/ 作为文章列表页
  • 文章详情继续使用默认 permalink

这样首页负责展示“我是谁、这里写什么、从哪里开始”，文章列表单独承担内容分发。

需要改哪些地方

1. 站点配置

在 _config.yml 里，把文章索引页路径改到独立路由：

2. 首页内容

自定义首页内容放在 source/index.md，它会直接生成站点根路径 /。

建议首页只承担三件事：

  • 自我介绍
  • 内容说明
  • 关键入口导航

3. 主题导航

导航建议直接围绕内容结构来设计：

  • 首页
  • 文章
  • 分类
  • 标签
  • 友链
  • 联系
  • 关于

比起在导航里放还没建设完成的页面，先把有效入口打通更重要。

4. 静态资源管理

自定义图片和图标尽量统一放在 source/img/，不要散落在主题目录里。

原因很直接：

  • 资源归属清晰
  • 换主题或升级主题时更安全
  • 链接路径更稳定

什么时候才需要改主题源码

大多数首页调整都可以通过下面两类文件完成：

  • _config.yml
  • _config.butterfly.yml

只有在你明确需要新增结构、改模板逻辑、改组件布局时，才建议进入 themes/butterfly/layout/ 或 themes/butterfly/source/。

先用配置解决问题，再进入模板层，这是维护成本最低的方式。
